# glyph ŌĆö SFTŌåÆRLVR for a Rust tool-use agent

Teaching `Qwen3-4B-Base` a coding-agent protocol on real Rust tasks, then probing
**where supervised learning ends and RL begins**. Tools execute for real against
sandboxed crates; `cargo_run` counts as success only if stdout exactly matches
expected, `cargo_test` only if tests pass ŌĆö the model cannot fake outcomes.

```text
assistant -> CALL ...   (read_file, apply_patch, cargo_test, cargo_run)
tool      -> RESULT ...
assistant -> FINAL: ...  (stop)
```

## Research question

SFT gives the model the capability (tool use, patching, multi-attempt recovery).
The single behavior it does *not* reliably learn is **terminating after success**:
it solves the task, then keeps using tools to the round cap and never emits
`FINAL`. Is that a data-coverage gap (fix with SFT) or a distribution-shift gap on
the model's own trajectories (fix with RL)? This repo runs that experiment.

## Findings

1. **SFT solves but won't stop.** SFT_V1: terminal 68/69, but 16 solved-but-no-FINAL.
2. **More/deeper recovery data does not fix stopping.** Across V1ŌåÆV2ŌåÆV3, clean_end
   plateaued at **0.72ŌĆō0.75**. Deeper-recovery data made the model *over-recover*
   (5 attempts where 1 suffices), trading which deep cases stop. ŌåÆ not a coverage gap.
3. **The first RLVR run collapsed** (`RLVR_V1`: 52ŌåÆ20) ŌĆö but from a broken reward:
   stacked ŌłÆ13..ŌłÆ23 penalties, no positive path on unsolved tasks, no SFT anchor
   (`teacher-tau 0.01`), ~56% zero-advantage groups. Not evidence RL can't do this.
4. **Conclusion:** stopping is distribution-shift on the model's *own* deep
   successes (SFT only ever shows the teacher's clean states) ŌåÆ an RL target. The
   corrected RL recipe is bounded, success-anchored, and anchored to SFT_V1.

## SFT training (SFT_V1, baseline recipe)

```bash
python -m sft.train \
  --model Qwen/Qwen3-4B-Base --tokenizer Qwen/Qwen3-4B-Base \
  --data synthetic_data/signal_1062.jsonl \
  --output runs/SIGNAL_1062_SFT_E3_LR2E5 \
  --epochs 3 --batch-size 1 --grad-accum 8 --lr 2e-5 \
  --max-seq-length 4096 --no-train-split
```

Deep-recovery datasets (v2/v3) need `--max-seq-length 11000 --gradient-checkpointing`
(a few traces reach ~10.4k tokens).

## Held-out eval

```bash
python -m sft.eval_formal \
  --sft-model <model-or-path> \
  --train-data synthetic_data/signal_1062.jsonl \
  --prompt-file sft/evals/eval_prompts_heldout_69.yaml --prompt-section post_eval_heldout_69 \
  --output <out>.json --max-new-tokens 4000 --max-tool-rounds 15 \
  --cases-root runs/rlvr1/rust_cases/eval_heldout_69
```

Contamination guard: `synthetic_data/audit_blueprint_similarity.py` (train vs eval at 0.92).

## RLVR

Reward (minimal, success-anchored; `rl/task_trace.py`): `+8` verifier passed,
`+3` passed-then-one-FINAL-no-tools-after, `-3` tool used after success (the
failure), `-2` round-cap, `┬▒1/2` finalize. One scalar per rollout. Recipe, gating,
and rationale: `rl/setup/rl_setup_training.md` and `rl/RLVR_NOTES.md`. Base = SFT_V1.
