# SFTâ†’RLVR for a Rust tool-use agent

Teaching `Qwen3-4B-Base` a coding-agent protocol on real Rust tasks, then probing **where supervised learning ends and RL begins**. Tools execute for real against sandboxed crates; `cargo_run` counts as success only if stdout exactly matches expected, `cargo_test` only if tests pass, the model cannot fake outcomes.

```text
assistant -> CALL ...   (read_file, apply_patch, cargo_test, cargo_run)
tool      -> RESULT ...
assistant -> FINAL: ...  (stop)
```

## Research question

SFT gives the model the capability (tool use, patching, multi-attempt recovery).
On the held-out set the one behavior it doesn't reliably do is **terminate after
success** â€” it solves, then churns to the round cap without `FINAL`. Is that an
SFT-coverage gap or an RL-fixable one? We ran the experiment, and the answer
**redirected the project**: churn is an OOD tail of the held-out set â€” train
prompts don't elicit it, so RL has no gradient to install stopping (two runs
collapsed; see Findings). RL's real lever here is **solve-rate on the cases the
policy only solves *sometimes*** (`0 < pass@k < k`). That's what this repo now
does: pass@k scan â†’ RL the partial-solve band â†’ measure the lift.

## Findings

1. **SFT solves but won't stop (on held-out).** SFT_V1: terminal 68/69, 16
   solved-but-no-FINAL. More/deeper recovery data didn't fix it (V1â†’V2â†’V3
   clean_end plateaued 0.72â€“0.75; deeper data made the model *over-recover*).
2. **RL can't install stopping â€” and that's the real finding.** Two stop-targeted
   variants both regressed full eval: `RLVR_V1` (stacked-penalty reward, 52â†’20)
   and `RLVR_B` (corrected bounded reward **+** `--terminal-on-success`, which
   forces the next turn terminal one step after the first pass, 52â†’19). Note B
   changes *two* things at once (reward and horizon truncation), so it is **not**
   a clean isolation of the reward â€” it just shows even the most aggressive
   credit trick for stopping still collapses. The load-bearing evidence is a
   **direct measurement**: the policy emits ~0 churn on train prompts (temp-0: 0
   churn; temp-0.8 depthâ‰¥3: 0/16). Churn is an **OOD tail of the held-out set**;
   it's not in the rollouts, so there's no gradient to shape â€” independent of
   reward or truncation. **You can't RL a behavior the policy never samples.**
3. **Where RL does help: solve-rate on the partial-solve band.** A pass@k scan
   (`sft/passk_scan.py`) bands train prompts: `0<pass@k<k` = RLVR-addressable
   (rollout variance = gradient), `==k` solved, `==0` capability-gap. RL on that
   band, verifier-dominant bounded reward, anchored to SFT_V1.
4. **Termination is an SFT-coverage problem**, parked for a future SFT pass that
   puts the hard held-out shapes in training so the model samples them.

## SFT training (SFT_V1, baseline recipe)

```bash
python -m sft.train \
  --model Qwen/Qwen3-4B-Base --tokenizer Qwen/Qwen3-4B-Base \
  --data synthetic_data/signal_1062.jsonl \
  --output runs/SIGNAL_1062_SFT_E3_LR2E5 \
  --epochs 3 --batch-size 1 --grad-accum 8 --lr 2e-5 \
  --max-seq-length 4096 --no-train-split
```

Deep-recovery datasets (v2/v3) need `--max-seq-length 11000 --gradient-checkpointing`
(a few traces reach ~10.4k tokens).

## Held-out eval

```bash
python -m sft.eval_formal \
  --sft-model <model-or-path> \
  --train-data synthetic_data/signal_1062.jsonl \
  --prompt-file sft/evals/eval_prompts_heldout_69.yaml --prompt-section post_eval_heldout_69 \
  --output <out>.json --max-new-tokens 4000 --max-tool-rounds 15 \
  --cases-root runs/rlvr1/rust_cases/eval_heldout_69
```

Pass `--sft-model <hf-or-path>` to eval any checkpoint (SFT or RLVR) the same way.

Contamination guard: `synthetic_data/audit_blueprint_similarity.py` (train vs eval at 0.92).

## RLVR (capability lift)

Pipeline: pass@k scan the train candidate set (`runs/rlvr_passk_train150/`) â†’ keep the `0<pass@k<k` rlvr-target band â†’ GRPO on that band â†’ measure in-set
pass@k lift, SFT_V1 vs best checkpoint.

Reward (`rl/task_trace.py`): verifier-dominant, **termination tails zeroed** for this run â€” `+8` verifier passed, `+0.5/âˆ’2/âˆ’1` format floor (valid / no-call / malformed), FINAL+churn+round-cap = `0`. One scalar per rollout; locked by `reward_golden_tests.py`. Full recipe, gating, and the lessons behind it: `rl/setup/rl_setup_training.md` and `rl/RLVR_NOTES.md`. Base = SFT_V1.
